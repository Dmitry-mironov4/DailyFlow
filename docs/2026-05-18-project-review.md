# DailyFlow — Полная оценка проекта
*Дата: 2026-05-18*

---

## 1. Резюме

DailyFlow — минималистичное iOS-приложение «всё в одном» (задачи + привычки + дневник настроения + инсайты) с тёплой карамельной тёмной дизайн-системой. Архитектура чистая, тест-покрытие сервисов отличное, дизайн выделяется на фоне конкурентов. Готовность к релизу: **~70–75%** — нужно устранить критические технические проблемы и добавить несколько must-have фич.

---

## 2. Конкурентный анализ

### Прямые конкуренты

| Приложение | Фокус | Цена | Что делает хорошо | Слабость |
|---|---|---|---|---|
| **Streaks** | Только привычки | $5.99 разово | HealthKit авто-трекинг, Apple-native дизайн, интерактивные виджеты | Нет задач, нет дневника, лимит 24 привычки |
| **Daylio** | Дневник + активности | $35.99/год | Минимальное трение (15 сек), богатая статистика, корреляции | Нет задач, нет focus task |
| **Things 3** | Только задачи | $9.99 разово | GTD-методология, золотой стандарт дизайна, natural language dates | Нет привычек, нет настроения, нет аналитики |
| **Habitica** | Привычки + задачи | Freemium | Геймификация, социальная ответственность, бесплатно | Перегружен, RPG-визуал, нет нативного iOS UX |
| **Pixels** | Дневник настроения | Freemium | Визуализация "год в пикселях", приватность, минимализм | Нет задач, нет streak analytics |
| **Bearable** | Health tracker | $34.99/год | Корреляции симптомов, медицинский контекст | Только для больных, нет задач |

### Позиция DailyFlow

**Уникальная ниша:** Единственное приложение, которое объединяет задачи дня + привычки + дневник + инсайты в **одном минималистичном тёмном интерфейсе** без подписочной перегрузки.

**Дифференциаторы, которых нет у конкурентов:**
- **Focus Task** (FocusCardView) — концепция «одна главная задача дня» нигде не реализована на уровне выделенного UI
- **RolloverBanner** — интеллектуальный перенос задач есть у единиц
- **Тёплая карамельная палитра** — выделяется на фоне серо-синих конкурентов

## 3. Сильные стороны текущей реализации

### Архитектура
- ✅ Чёткое разделение Models / Services / Views
- ✅ Enum-namespace Services — stateless, тестируемы без моков
- ✅ `@Query` + SwiftData — правильное использование без лишних ObservableObject
- ✅ Дизайн-токены через ColorExtensions и ViewExtensions — консистентность
- ✅ Cross-midnight через `.id(dateAnchor)` — элегантное решение

### Качество кода
- ✅ Покрытие тестами сервисов: ~95% (53 теста, включая граничные случаи)
- ✅ Размер файлов Views в основном ≤ 150 строк
- ✅ Haptic feedback через `Haptics.tap()`
- ✅ `#Preview` в каждой View с реалистичными сценариями

### Фичи
- ✅ WidgetKit + AppIntents (интерактивный toggle задач с Home Screen)
- ✅ EventKit синхронизация задач с Календарём
- ✅ PixelGrid для 7-дневной визуализации привычек
- ✅ Swift Charts для графика настроения

## 4. Технические проблемы

### КРИТИЧЕСКИЕ — исправить до релиза

#### 4.1 Дублирование модели DailyTask
```
DailyFlow/Models/DailyTask.swift
DailyFlowWidgets/DailyTask+Widget.swift  ← идентичная копия!
```
При любом изменении модели нужно менять два файла. Риск десинхронизации.
**Решение:** Вынести модели в отдельный Swift Package или использовать App Group shared framework.

#### 4.2 Дублирование цветов в виджете
```swift
// DailyFlowWidgets.swift — копия токенов из ColorExtensions
private extension Color {
    static let wBgPrimary = Color(hex: 0x0D0A05)
    static let wAccentTeal = Color(hex: 0xD4882A)
    ...
}
```
**Решение:** То же — shared framework для токенов.

#### 4.3 Порядок операций в TaskService — потенциальное рассогласование
```swift
// TaskService.add()
CalendarService.sync(task, ctx: ctx)  // ← вызов ПЕРЕД save()
try? ctx.save()                        // ← если упадёт — событие в Calendar есть, задачи нет
```
**Решение:** Переместить `CalendarService.sync()` ПОСЛЕ `ctx.save()`.

#### 4.4 Хардкод длительности события в Calendar
```swift
// CalendarService.swift:25
event.endDate = scheduledTime.addingTimeInterval(3600) // жёстко 1 час!
```
**Решение:** Добавить `duration: TimeInterval` в сигнатуру `sync()`, дефолт 3600.

---

### СЕРЬЁЗНЫЕ — важны для UX

#### 4.5 Молчаливое проглатывание ошибок (~20 мест)
```swift
try? ctx.save()    // ошибка сохранения проглочена
try? ctx.fetch()   // ошибка fetch проглочена
```
Пользователь не знает, что данные не сохранились.
**Решение:** Добавить хотя бы `os_log` для ошибок. В будущем — Toast-уведомление.

#### 4.6 `precondition` в JournalService — крэш в runtime
```swift
func setMood(_ score: Int, ...) {
    precondition((1...5).contains(score)) // ← крэш при невалидных данных
}
```
При этом сама модель `JournalEntry.moodScore: Int` не валидирует диапазон.
**Решение:** Заменить `precondition` на `guard` с ранним выходом или `Result`.

#### 4.7 Хардкод цветов настроения — нарушение дизайн-системы
```swift
// MoodPickerView.swift — цвета не из ColorExtensions!
case 1: return Color(hex: 0xFF6B6B)  // красный
case 5: return Color.accentTeal      // а здесь уже токен
```
**Решение:** Добавить `moodColor(for score: Int)` в ColorExtensions.

#### 4.8 Реактивность InsightsContentView через хак
```swift
var dataChangeToken: Int {
    allTasks.count &+ allHabits.count &+ allEntries.count  // &+ overflow-aware
}
```
Любое изменение данных (добавление задачи) триггерит пересчёт всех инсайтов.
**Решение:** Передавать данные через `@Query` напрямую в `InsightsService`, убрать `dataChangeToken`.

#### 4.9 DateFormatter создаётся на каждый рендер
```swift
// TodayContentView — computed property создаёт DateFormatter каждый раз
var dateCaption: String {
    let formatter = DateFormatter()  // ← дорогой объект!
    ...
}
```
**Решение:** `static let` форматтер в extension или кэш в `@State`.

### УМЕРЕННЫЕ — улучшение качества

#### 4.10 Только 3 цвета для привычек
```swift
// AddHabitSheet.swift
let colorOptions = ["2DD4A0", "F0A23B", "9B8AE8"]  // жёстко 3 цвета
```

#### 4.11 O(n×m) в `InsightsService.habitsRate()`
Для каждого дня (7) × каждая привычка (n) × логи (m). На 50+ привычках заметно.
**Решение:** Предварительная индексация логов через `Dictionary(grouping:)`.

```swift

```

## 5. Что добавить — сравнение с конкурентами

### MUST-HAVE (по данным рынка — без этого плохие отзывы)

| Фича | Конкуренты | Статус DailyFlow | Приоритет |
|---|---|---|---|
| **Локальные уведомления** | Все | ❌ Не реализованы | 🔴 Критично |
| **Напоминание о привычках** | Streaks, Habitica | ❌ Нет | 🔴 Критично |
| **Экспорт данных** | Bearable, Daylio | ⚠️ Obsidian в TODO | 🔴 Критично |
| **Lock screen виджет** | Streaks, Daylio | ❌ Нет | 🟡 Важно |

### NICE-TO-HAVE (дифференциация)

#### 5.1 Корреляции настроения и привычек
Killer feature Daylio и Bearable. Пользователи хотят видеть «что влияет на моё настроение».
```
Пример: "В дни, когда ты делаешь Спорт, настроение +1.2 балла"
```
Данные уже есть в HabitLog + JournalEntry — нужна только аналитика.

#### 5.2 Grace Period для стриков
Главная причина, почему пользователи бросают habit apps — сломанный стрик.
```
"Streak Shield": 1 пропуск в 2 недели не сбрасывает стрик
```

#### 5.3 Вечерний recap / утренняя сводка
Уведомление «Сегодня выполнено 3/5 задач, стрик Спорт 🔥7» — высокое retention.
Уже частично реализовано через `rolloverPending`.

#### 5.4 HealthKit интеграция (авто-трекинг)
Streaks выигрывает здесь. Шаги, сон, медитация — автоматически без ручного отмечания.

#### 5.5 Экспорт в CSV
Obsidian — нишевый экспорт. CSV для врача / таблиц востребован у Bearable-аудитории.

#### 5.6 Кастомный emoji/название для уровней настроения
Daylio позволяет переименовывать «Ужасно» → своё название. Персонализация = retention.

#### 5.7 AI-инсайты (быстро становится must-have к 2026)
```
"Твоя продуктивность на 40% выше в будни до полудня"
"Стрик Медитация растёт — ты на 5-й день подряд"
```

## 6. UX-улучшения

### 6.1 Onboarding
Ни один конкурент не делает onboarding хорошо. Шанс выделиться:
- 3 экрана: «Выбери свой фокус на сегодня», «Добавь первую привычку», «Как ты себя чувствуешь?»
- Создание демо-данных при первом запуске

### 6.2 Celebration micro-animations
Haptic есть — визуальной анимации нет. При закрытии задачи / завершении привычки:
- Короткая particle animation (без тяжёлых библиотек — SwiftUI `Canvas`)
- Символ ✓ с scale + opacity animation

### 6.3 Empty state экраны
При первом запуске экраны пустые — нет подсказки что делать. 
Добавить иллюстративный текст + CTA-кнопку для каждого пустого экрана.

### 6.4 Quick entry через Shortcuts / Siri
Things 3 и Streaks поддерживают «Добавить задачу» через Siri. 
AppIntents уже используются для виджетов — расширить на Shortcuts.

## 8. Итоговая оценка

| Категория | Оценка | Комментарий |
|---|---|---|
| Архитектура | 8/10 | Чистая, но дублирование с виджетом нужно решить |
| Код quality | 7/10 | Хороший, но silent errors и hardcodes снижают |
| Тест покрытие | 8/10 | Сервисы отличные, Views — 0% |
| Дизайн | 9/10 | Уникальная палитра, консистентная система |
| Фичи vs конкуренты | 6/10 | Нет уведомлений, нет экспорта, нет корреляций |
| Дифференциация | 9/10 | Focus Task + RolloverBanner + дизайн — сильно |
| **Итого** | **7.8/10** | Крепкий фундамент, нужна финальная полировка |

**Главный вывод:** DailyFlow занимает реально незаполненную нишу — all-in-one без перегрузки с фокусом на «главное дело дня». Технически почти готов. Три вещи, которые разблокируют релиз: уведомления, экспорт данных, исправление порядка calendar sync.
